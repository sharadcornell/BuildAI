Create a new full responsive website prototype for **BuildAI**.

Do not design this like a generic AI SaaS website, bootcamp landing page, dark-mode dashboard site, or hacker terminal page.

The concept should be:

**BuildAI = a live engineering apprenticeship floor.**

The website should feel like the visitor is entering a real cohort command center where students are shipping products every week, mentors are reviewing work, colleges can see progress, and demo day is always approaching.

The design should combine:

1. **Cohort Command Center** — live operational view of a 13-week apprenticeship.
2. **Engineering Lab Notebook** — mentor notes, build logs, weekly artifacts, project evidence, certification stamps.
3. **Premium education trust** — credible enough for college decision-makers and TPOs.
4. **Student-builder energy** — exciting enough for ambitious engineering students.

Primary audiences:

* Colleges and TPOs
* Engineering students

Secondary audiences:

* Mentors
* Future hiring partners

Brand:

* Name: BuildAI
* Positioning: A 13-week AI product-engineering apprenticeship run inside India’s engineering colleges.
* Core message: Students do not just learn AI. They ship real products, get mentor-reviewed, and build portfolio proof.
* Do not promise jobs.
* Do not use fake logos.
* Do not show exact pricing.
* Use trust-safe language for placements: “opportunities, not guarantees.”

Visual direction:

* Avoid generic cyan-gradient AI SaaS visuals.
* Avoid a giant normal hero with a dashboard floating on the right.
* Avoid generic glowing blobs, rounded SaaS cards, and overused terminal-only visuals.
* Use a distinctive visual system based on:

  * cohort status panels
  * live build logs
  * weekly sprint boards
  * mentor review notes
  * demo day countdowns
  * student project evidence cards
  * certification stamps
  * college progress reports
  * lab notebook annotations
  * pinned critique notes
  * shipping checklists
  * rubric scorecards
* Make it feel premium, not childish or chaotic.
* It should feel like “a selective engineering apprenticeship under real shipping pressure.”

Color and typography:

* Base: deep ink / near-black / dark navy.
* Surfaces: warm off-white, soft paper, muted slate.
* Accent: one strong signal color like safety orange or electric coral.
* Secondary accents: muted mint/green only for status indicators.
* Use bold condensed typography for major headlines.
* Use clean sans-serif for body copy.
* Use mono text only for labels, build logs, timestamps, and system status.
* Keep spacing disciplined and grid-based.
* Use some rough/lab-notebook elements, but keep the layout polished.
